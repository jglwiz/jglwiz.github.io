Debug the build of pelican blog

Posted on Fri 02 June 2017 in pelican

This post offers a few additional explanation to this tutorial:Building a data science portfolio, which provides useful infommation for a beginner to start a pelican blog from scratch on github pages.

1. Repos' structure

The original reading suggests the use of ghp-import, to make the blog publishing program and output in the same github repo. (master branch for output folder, dev branch for publishing program).

A better solution is to put them into different github repos. Put the blog publishing program into blog repo, and the output folder into username.github.io repo as submodule of blog repo.

1) Create two repositories in Github with names such as username.github.io and blog

2) Setup your full blog folder in the blog repo (git@github.com:username/blog.git)

cd your_blog_folder
git remote add origin git@github.com:username/blog.git

3) Setup your output folder using username.github.io (git@github.com:username/username.github.io.git)

git submodule add git@github.com:username/username.github.io.git output

4) change the config file in publishconf.py otherwise the output folder would be deleted each time runing 'make publish` by default

# DELETE_OUTPUT_DIRECTORY = True
DELETE_OUTPUT_DIRECTORY = False

5) check the submodule. The following command will show git remote url.

check the root folder: 

cd blog
git remote show origin

check the output folder:

cd output
git remote show origin
